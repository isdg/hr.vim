*hr.txt*  A reading-list sidebar backed by the `hr` CLI       *hr* *hr.vim*

==============================================================================
INTRODUCTION                                                    *hr-intro*

hr.vim is a thin sidebar over the `hr` reading-list CLI. It lists articles,
opens them in the main window, and maps a few keys to mark read/unread,
favorite, alias, sync, and refresh. Pure Vimscript — works in Vim 8.1+ and
Neovim.

==============================================================================
CONFIGURATION                                                   *hr-config*

Set these globals before the plugin loads (or any time; they are read live):

                                                                *g:hr_binary*
g:hr_binary     CLI name or path.                       Default: "hr"

                                                                *g:hr_vault*
g:hr_vault      Vault directory. When empty, the vault is read from the
                `vault = "..."` line in ~/.hrrc and the plugin :cd's into it.
                When set, it is passed to the CLI as `-C <vault>`.
                                                        Default: "" (use rc)

                                                                *g:hr_side*
g:hr_side       Sidebar side: "left" or "right".        Default: "left"

                                                                *g:hr_width*
g:hr_width      Sidebar width in columns.               Default: 60

                                                                *g:hr_show_read*
g:hr_show_read  1 to include read items, 0 for unread only.
                                                        Default: 1

                                                            *g:hr_corrupt_maps*
g:hr_corrupt_maps   1 to install the buffer-local corruption-marking keys on
                    articles opened from the panel, 0 to bind them yourself
                    via the <Plug> maps below.           Default: 1

                                                          *g:hr_corrupt_prefix*
g:hr_corrupt_prefix Key prefix for those buffer-local maps.
                                                        Default: "<leader>n"

==============================================================================
COMMANDS                                                        *hr-commands*

:Hr  :HrToggle          Toggle the sidebar. When opening, closes all other
                        windows so the feed is the only one (modified buffers
                        are hidden, not lost). Use :HrOpen to keep them.
:HrOpen                 Open the sidebar (focus it if already open). Leaves
                        existing windows untouched.
:HrClose                Close the sidebar.
:HrStart                Open panel only; close the initial empty [No Name]
                        window if present. Entry point for the `hr` CLI.
:HrRefresh              Re-fetch and redraw the list.
:HrSync                 Run `hr sync`, then refresh.

                                                            *hr-corrupt-commands*
Run these from within an opened article (any saved .md buffer):

:[range]HrCorrupt [note]  Mark the selected region as corrupted (an LLM can
                        repair it later via `hr corrupt restore`). Use it from
                        a visual selection — `:'<,'>HrCorrupt OCR garble`. The
                        exact selection is sent to the CLI, which rejects the
                        mark if it doesn't match the computed byte range.
:HrCorruptUndo          Remove the most recent corruption mark on the article.
:HrLocate               Jump the cursor to this article's row in the feed
                        panel (opening the panel if closed), so you can act on
                        it there. See |hr-locate|.

==============================================================================
CORRUPTION MARKING                                              *hr-corrupt*

Select garbled text in an article and mark it so tooling can restore it from
the stored quote + surrounding context. Marks live in the article's sidecar;
`hr corrupt list --all --json` is the unified report a repair tool consumes.

Buffer-local keys on articles opened from the panel (see |g:hr_corrupt_maps|,
|g:hr_corrupt_prefix|; shown with the default "<leader>n" prefix):

  <leader>nc   (visual) mark the selection corrupted
  <leader>nn   (visual) mark the selection, prompting for a note
  <leader>nu   (normal) undo the most recent mark

To bind your own keys, map the <Plug> targets (available everywhere): >
    xmap <leader>x  <Plug>(HrCorrupt)
    xmap <leader>X  <Plug>(HrCorruptNote)
    nmap <leader>xu <Plug>(HrCorruptUndo)
<

==============================================================================
LOCATE IN FEED                                                  *hr-locate*

From any saved article buffer — however it was opened — |:HrLocate| (or the
<Plug>(HrLocate) target) checks whether the file belongs to the feed. If it
does, the panel is opened (when closed) and the cursor jumps to the article's
row, so you can manage it there — mark read, favorite, alias — without
scrolling to find it. If it does not, nothing happens beyond a message; no
panel is opened. Membership is checked before any window is touched.

No key is bound by default; bind the <Plug> target yourself, e.g.: >
    nmap <leader>nf <Plug>(HrLocate)
<
Membership is checked against the full list, so a read article is found even
under an unread-only view (|g:hr_show_read| is 0). Locating a read one in that
case turns |g:hr_show_read| on for the session, since the cursor can only land
on a row the panel actually renders.

==============================================================================
SIDEBAR KEYS                                                    *hr-keys*

Active only inside the sidebar buffer:

  <CR>, o     open the article under the cursor
  r           toggle read / unread
  u           mark unread
  f           toggle favorite
  a           set alias (rename the displayed label)
  R           refresh
  s           sync + refresh
  q           close panel
  ?           show this key list

==============================================================================
EXAMPLE                                                         *hr-example*

Neovim (Lua): >
    vim.g.hr_width = 50
    vim.keymap.set("n", "<leader>r", "<Cmd>HrToggle<CR>",
       { desc = "Toggle hr reading list" })
<
Vim (Vimscript): >
    let g:hr_width = 50
    nnoremap <leader>r <Cmd>HrToggle<CR>
<
